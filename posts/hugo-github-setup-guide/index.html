今天正式把博客搭好了！虽然中间遇到了不少“红字”报错，但解决问题的过程真的很有成就感。

遇到的坑与解决方法

1. Git Push 报错（Token 权限）

现象：输入账号密码后报错，提示不支持密码认证。 解决：GitHub 现在需要使用 Personal Access Token (PAT)。

  • 关键点：一定要勾选 repo 和 workflow 权限，否则没法上传自动化脚本。

2. 自定义域名消失术

现象：在 GitHub 后台设好了域名，一推新文章域名就没了。 解决：在本地 static/ 文件夹下创建 CNAME 文件，写上自己的域名。这样每次自动化部署都会带上它。

3. 访问 404 报错

现象：手机能开，电脑打不开。 解决：这是“浏览器缓存”的锅。清理缓存或者换个浏览器就好了。

4. 样式乱码

现象：网页只有文字，没有排版。 解决：修改 hugo.toml 里的 baseURL 为自己的真实域名 https://wanger.cc/。

总结

搭建博客不只是为了写字，更是为了学会一套“自动化发布”的工作流。感谢老师的指导，下一步准备研究下如何优化 Archie 主题！
