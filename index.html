Hugo Github Setup Guide

Dec 28, 2025

title: “从零开始：我的 Hugo + GitHub 博客搭建踩坑记” date: 2025-12-28 draft: false tags: [“Hugo”, “GitHub”, “折腾记录”]

今天正式把博客搭好了！虽然中间遇到了不少“红字”报错，但解决问题的过程真的很有成就感。

遇到的坑与解决方法

1. Git Push 报错（Token 权限）

现象：输入账号密码后报错，提示不支持密码认证。 解决：GitHub 现在需要使用 Personal Access Token (PAT)。

  • 关键点：一定要勾选 repo 和 workflow 权限，否则没法上传自动化脚本。

2. 自定义域名消失术

现象：在 GitHub 后台设好了域名，一推新文章域名就没了。 解决：在本地 static/ 文件夹下创建 CNAME 文件，写上自己的域名。这样每次自动化部署都会带上它。

3. 访问 404 报错

现象：手机能开，电脑打不开。 解决：这是“浏览器缓存”的锅。清理缓存或者换个浏览器就好了。

4. 样式乱码

现象：网页只有文字，没有排版。 解决：修改 hugo.toml 里的 baseURL 为自己的真实域名 https://wanger.cc/。

总结

搭建博客不只是为了写字，更是为了学会一套“自动化发布”的工作流。感谢老师的指导，下一步准备研究下如何优化 Archie 主题！

发布时间：2025-12-28 | 作者：我 | 标签：新手入门、博客记录

一、为什么想写博客？

作为一个编程新手，我一直想找一个地方记录自己的学习过程：

  • 把学到的知识点整理成文字，加深记忆
  • 分享自己踩过的坑，帮助和我一样的新手
  • 慢慢积累，见证自己的成长

二、接下来我想分享什么？

目前我主要在学习前端和终端操作，接下来会分享这些内容：

  1. Mac终端的常用命令（比如cd、find、clear）
  2. Markdown的基础用法（就是现在写博客用的语法）
  3. 搭建个人博客的小技巧

三、一个小彩蛋：我学会的第一个终端命令

今天刚学会用 cd 命令进入博客文件夹，步骤很简单：

  1. 打开终端
  2. 用 cd 文件夹路径 进入存放博客的目录
  3. 用 pwd 验证是否进入成功
